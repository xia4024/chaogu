actual_trades 表设计
字段类型说明
idINT PK
account_idINT FK多账号隔离
symbolVARCHAR(20)证券代码
nameVARCHAR(100)证券名称
actionVARCHAR(10)buy / sell
trade_dateDATE成交日期 ⚡精确
trade_timeTIME成交时间
priceDECIMAL(10,3)成交价格
sharesINT成交数量
amountDECIMAL(14,2)成交金额
commissionDECIMAL(10,2)手续费（如有）
image_idINT FK🆕 来源图片
ocr_confidenceDECIMAL(4,3)🆕 OCR 置信度
ocr_raw_textTEXT🆕 OCR 原始输出
sourceVARCHAR(20)🆕 ocr / manual
notesTEXT备注
created_atDATETIME
trade_images 表 & 存储
字段说明
idINT PK
account_id所属账号
file_path本地路径 uploads/trades/
file_hashMD5 去重，防止重复上传
trade_date_hint用户标注的日期（用于历史补录）
ocr_statuspending / done / failed
ocr_result_jsonOCR 结构化结果 JSON
uploaded_atDATETIME
📁 存储策略
uploads/trades/{account_id}/{YYYY-MM}/
按账号 + 月份分目录，便于管理
结构化解析策略
1. PaddleOCR 输出 → 文本框列表 [{text, bbox, confidence}]
2. 排版聚类 → 按 Y 坐标分组，识别每行记录
3. 正则匹配 → 代码 \d{6} | 价格 \d+\.\d+ | 买/卖 | 日期 \d{4}-\d{2}-\d{2}
4. 字段映射 → 将匹配结果填入 actual_trades 字段
