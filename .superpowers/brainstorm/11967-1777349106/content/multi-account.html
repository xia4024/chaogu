多账号架构设计 — 数据隔离方案

当前持仓是一个账号，后续可能增加其他账号

A

简单方案：accounts 表 + account_id 外键

所有现有表加 account_id 列，同一数据库内按账号隔离。配置文件中每个账号独立配持仓。

适用：1-3 个账号 | 改动量小 | 查询需带 account_id
B

中等方案：独立 schema / database

每个账号独立 MySQL database 或 schema，物理隔离。配置文件支持多账号切换。

适用：账号间需完全隔离 | 备份恢复方便 | 跨账号查询困难
C

推荐：account_id 逻辑隔离 + 账号管理页

accounts 表管理多账号（名称/券商/类型），所有数据表加 account_id。Web 页面顶部加账号切换器，当前上下文自动过滤。一个 account_id=0 作为"策略模拟"账号。

适用：你的场景 | 改动适中 | 跨账号对比可行 | 与现有一致
