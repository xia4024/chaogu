数据库设计和页面结构

MySQL 表设计 + Flask 前端页面规划

MySQL 表设计（3 张表）

scan_reports — 每次扫描记录
字段 类型 说明
idINT AUTO_INCREMENT主键
scan_dateDATE扫描日期
benchmark_symbolVARCHAR(20)基准标的
market_regimeVARCHAR(20)市场状态 risk_on/risk_off
candidate_countINT候选信号数
filtered_countINT过滤标的数
created_atDATETIME入库时间
trade_plans — 交易计划（一对多关联 scan_reports）
字段 类型 说明
idINT AUTO_INCREMENT主键
report_idINT关联 scan_reports.id
strategy_idVARCHAR(50)策略ID
signal_groupVARCHAR(20)open / t_trade
actionVARCHAR(10)buy / sell
symbol / name / score-标的、名称、评分
entry_price / stop_loss / take_profitDECIMAL(10,3)参考价/止损/止盈
suggested_shares / suggested_valueINT / DECIMAL(12,2)建议股数/金额
reasons / tagsTEXTJSON 格式存储
holdings — 持仓管理（独立表，与扫描无外键关联）
字段 类型 说明
idINT AUTO_INCREMENT主键
symbolVARCHAR(20) UNIQUEETF 代码
sharesINT持仓股数
cost_basisDECIMAL(10,3)成本价
min_t_trade_pctDECIMAL(4,3)最小T比例
max_t_trade_pctDECIMAL(4,3)最大T比例
updated_atDATETIME更新时间

前端页面规划

扫描记录列表

首页。按日期倒序展示所有扫描记录，卡片式布局，每条显示日期、市场状态、信号统计摘要。点击进入详情

扫描详情

展示单次扫描的全部交易计划，分"开仓信号"和"做T信号"两个区域，表格展示含评分、价格、建议仓位、触发原因

持仓管理

列表展示当前所有持仓，支持新增/编辑/删除。每行显示代码、名称、股数、成本价、T比例。附填写说明
